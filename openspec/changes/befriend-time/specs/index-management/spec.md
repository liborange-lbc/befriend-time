## 新增需求

### 需求:用户可以录入指数
系统必须允许用户录入新的指数信息，包括指数名称、代码和区域。

#### 场景:录入新指数
- **当** 用户在指数管理页面点击"添加指数"
- **那么** 系统显示录入表单，包含名称、代码、区域三个字段
- **当** 用户填写完整信息并提交
- **那么** 系统验证代码格式，保存到 indices.json，显示成功提示

#### 场景:代码格式验证
- **当** 用户输入的代码不符合格式（如缺少后缀）
- **那么** 系统显示错误提示，阻止保存

### 需求:用户可以编辑指数
系统必须允许用户修改已录入的指数信息。

#### 场景:编辑指数信息
- **当** 用户在指数列表中点击"编辑"
- **那么** 系统显示预填充的编辑表单
- **当** 用户修改信息并保存
- **那么** 系统更新 indices.json，显示成功提示

### 需求:用户可以删除指数
系统必须允许用户删除不再需要的指数。

#### 场景:删除指数
- **当** 用户在指数列表中点击"删除"
- **那么** 系统显示删除确认对话框
- **当** 用户确认删除
- **那么** 系统从 indices.json 移除该指数，同时保留历史数据文件

### 需求:用户可以查看指数列表
系统必须展示所有已录入的指数列表。

#### 场景:查看指数列表
- **当** 用户进入指数管理页面
- **那么** 系统显示所有指数列表，包含名称、代码、区域、数据状态
- **当** 指数数据有更新
- **那么** 列表显示最后更新时间

### 需求:指数区域分类
系统必须支持中/港/美三个区域的指数分类。

#### 场景:区域选择
- **当** 用户录入或编辑指数
- **那么** 系统提供区域下拉选择：中国(CN)、香港(HK)、美国(US)
- **当** 用户选择区域
- **那么** 系统记录区域信息，用于数据获取时区判断

### 需求:指数代码格式
系统必须支持主流指数代码格式。

#### 场景:代码格式校验
- **当** 用户输入代码
- **那么** 系统验证格式为：'{代码}.{交易所后缀}'
- **当** 交易所后缀为 SH/SZ（中国）、HK（香港）
- **那么** 系统识别为相应区域，无需手动选择区域
- **当** 代码不含交易所后缀
- **那么** 系统提示用户补充完整格式
