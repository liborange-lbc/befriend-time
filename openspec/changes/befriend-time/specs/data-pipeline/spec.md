## 新增需求

### 需求:系统必须定时获取收盘价数据
系统必须通过 GitHub Actions 定时从 Tushare API 获取指数收盘价数据。

#### 场景:A股数据获取
- **当** 北京时间到达 17:00
- **那么** 系统获取所有区域为 CN 的指数当日收盘价数据
- **当** 获取成功
- **那么** 系统保存数据到对应 JSON 文件，更新 update-log.json

#### 场景:港股数据获取
- **当** 北京时间到达 18:00
- **那么** 系统获取所有区域为 HK 的指数当日收盘价数据
- **当** 获取成功
- **那么** 系统保存数据到对应 JSON 文件，更新 update-log.json

#### 场景:美股数据获取
- **当** 北京时间到达次日 06:00
- **那么** 系统获取所有区域为 US 的指数当日收盘价数据
- **当** 获取成功
- **那么** 系统保存数据到对应 JSON 文件，更新 update-log.json

### 需求:系统必须计算多周期SMA均值
系统必须为每条收盘价数据计算 30/60/90/180/240/360 日简单移动平均线。

#### 场景:SMA计算
- **当** 系统获取到新的收盘价数据
- **那么** 系统计算该日期的所有周期SMA值
- **当** 数据天数少于SMA周期
- **那么** 该周期SMA值为 null

#### 场景:SMA计算公式
- **当** 计算 N 日 SMA
- **那么** 值为最近 N 个交易日收盘价的简单算术平均值

### 需求:系统必须计算均值偏差
系统必须计算每个收盘价相对于各周期均值的绝对偏差。

#### 场景:偏差计算
- **当** 系统计算出所有周期的SMA值
- **那么** 系统计算各周期偏差：偏差 = 收盘价 - SMA
- **当** SMA为 null
- **那么** 该周期偏差为 null

### 需求:系统必须支持数据初始化
系统必须支持首次运行时获取全量历史数据。

#### 场景:首次初始化
- **当** 某指数的年度数据文件不存在
- **那么** 系统从 Tushare 获取该指数从当前日期向前到最早可获取日期的所有历史数据
- **当** 历史数据获取完成
- **那么** 系统按年份分片存储到 JSON 文件

### 需求:系统必须支持增量更新
系统必须支持日常运行时获取近期数据覆盖更新。

#### 场景:日常增量更新
- **当** 每日定时任务执行
- **那么** 系统获取每个指数近 7 个交易日的数据
- **当** 新数据与已有数据日期相同
- **那么** 系统覆盖旧数据（防止数据错误）

### 需求:系统必须处理获取失败
系统必须在数据获取失败时进行重试。

#### 场景:失败重试
- **当** 数据获取失败
- **那么** 系统在 30 分钟后重试
- **当** 重试次数未达 10 次
- **那么** 系统继续重试
- **当** 重试次数达到 10 次仍失败
- **那么** 系统记录失败状态到 update-log.json，不再重试

#### 场景:失败状态记录
- **当** 获取失败
- **那么** 系统在 update-log.json 记录失败原因、重试次数
- **当** 获取最终成功
- **那么** 系统更新状态为成功，清除重试次数

### 需求:系统必须安全存储API密钥
系统必须将 Tushare API 密钥存储在 GitHub Secrets 中。

#### 场景:API密钥配置
- **当** 用户配置数据源
- **那么** 系统引导用户将 API Key 配置到仓库的 GitHub Secrets
- **当** GitHub Actions 执行
- **那么** 系统从环境变量读取 API Key，不在代码中硬编码

### 需求:数据必须按年份分片存储
系统必须将指数数据按年份分片存储为独立 JSON 文件。

#### 场景:年度数据存储
- **当** 系统保存指数数据
- **那么** 系统按 {代码}-{年份}.json 格式存储文件
- **当** 某年数据已存在
- **那么** 系统追加或覆盖该年份的数据
