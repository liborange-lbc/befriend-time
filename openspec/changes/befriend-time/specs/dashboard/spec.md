## 新增需求

### 需求:Dashboard必须展示策略信号
系统必须在Dashboard顶部展示所有已保存策略的当前信号。

#### 场景:策略信号展示
- **当** 用户进入Dashboard
- **那么** 系统在最上方展示所有已保存策略的综合信号
- **当** 某指数触发买入信号
- **那么** 显示买入提示，包含指数名称、策略名称、当前偏差
- **当** 某指数触发卖出信号
- **那么** 显示卖出提示，包含指数名称、策略名称、当前偏差

#### 场景:无信号状态
- **当** 没有任何指数触发策略信号
- **那么** 显示"当前无触发信号"提示

### 需求:Dashboard必须展示数据更新状态
系统必须展示各区域数据的最后更新时间和状态。

#### 场景:更新状态展示
- **当** 用户进入Dashboard
- **那么** 系统显示A股、港股、美股的最后更新时间
- **当** 数据更新成功
- **那么** 显示绿色状态标识和更新时间
- **当** 数据更新失败
- **那么** 显示红色状态标识、失败原因和重试次数

### 需求:Dashboard必须展示偏差监控
系统必须展示各指数当前偏差的一览视图。

#### 场景:偏差一览
- **当** 用户进入Dashboard
- **那么** 系统展示所有指数的当前偏差值（相对于60日均值）
- **当** 偏差值较大（如 > 5%）
- **那么** 使用红色或暖色标识
- **当** 偏差值较小或为负（如 < -5%）
- **那么** 使用绿色或冷色标识

#### 场景:偏差排序
- **当** 用户点击偏差列标题
- **那么** 系统按偏差值升序或降序排列指数列表

### 需求:Dashboard必须提供快捷入口
系统必须提供快速跳转到常用功能的入口。

#### 场景:快捷入口
- **当** 用户进入Dashboard
- **那么** 系统显示快捷入口：数据分析、策略回测、指数管理、策略管理
- **当** 用户点击某个入口
- **那么** 跳转到对应页面

### 需求:Dashboard必须展示最近查看的指数
系统必须展示用户最近查看过的指数。

#### 场景:最近查看
- **当** 用户进入Dashboard
- **那么** 系统显示最近查看过的 3-5 个指数
- **当** 用户点击某个指数
- **那么** 跳转到该指数的数据分析页面

### 需求:Dashboard界面必须科技感强
系统Dashboard界面必须呈现科技感风格。

#### 场景:科技感设计
- **当** 用户进入Dashboard
- **那么** 页面呈现浅色主题、简洁线条、现代字体
- **当** 信息密集展示
- **那么** 保持清晰可读，不过分拥挤
