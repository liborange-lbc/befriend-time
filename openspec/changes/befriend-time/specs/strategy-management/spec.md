## 新增需求

### 需求:用户可以保存策略
系统必须允许用户保存策略配置以便复用。

#### 场景:保存策略
- **当** 用户配置好策略参数
- **那么** 用户可以为策略命名并保存
- **当** 保存成功
- **那么** 策略存储到 strategies.json，显示成功提示

### 需求:用户必须为策略命名
系统保存策略时必须要求用户提供策略名称。

#### 场景:策略命名
- **当** 用户保存策略
- **那么** 系统要求输入策略名称
- **当** 用户输入名称并确认
- **那么** 系统保存策略及其名称

### 需求:用户可以查看策略列表
系统必须展示所有已保存的策略列表。

#### 场景:查看策略列表
- **当** 用户进入策略管理页面
- **那么** 系统显示所有已保存策略的列表
- **当** 列表为空
- **那么** 显示空状态提示

### 需求:用户可以编辑已保存的策略
系统必须允许用户修改已保存策略的参数。

#### 场景:编辑策略
- **当** 用户在策略列表中点击"编辑"
- **那么** 系统显示预填充的策略参数表单
- **当** 用户修改参数并保存
- **那么** 系统更新 strategies.json，显示成功提示

### 需求:用户可以删除策略
系统必须允许用户删除不再需要的策略。

#### 场景:删除策略
- **当** 用户在策略列表中点击"删除"
- **那么** 系统显示删除确认对话框
- **当** 用户确认删除
- **那么** 系统从 strategies.json 移除该策略

### 需求:用户可以快速应用已保存策略
系统必须支持从策略列表快速应用到回测。

#### 场景:快速应用策略
- **当** 用户在策略列表中点击"应用"
- **那么** 系统跳转到策略回测页面，自动填充该策略的参数
- **当** 用户选择指数和时间范围
- **那么** 可以直接运行回测
