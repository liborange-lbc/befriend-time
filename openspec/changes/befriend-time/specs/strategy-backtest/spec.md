## 新增需求

### 需求:系统必须支持均值偏差策略回溯
系统必须支持基于偏差阈值的策略回溯。

#### 场景:配置均值偏差策略
- **当** 用户选择"均值偏差策略"
- **那么** 系统显示参数配置：均值周期、买入阈值、卖出阈值、买入比例、卖出比例、初始资金
- **当** 用户配置参数并运行回溯
- **那么** 系统根据历史数据模拟交易

#### 场景:均值偏差买入信号
- **当** 偏差值低于买入阈值（如 -5%）
- **那么** 系统触发买入信号，按买入比例建仓

#### 场景:均值偏差卖出信号
- **当** 偏差值高于卖出阈值（如 +5%）
- **那么** 系统触发卖出信号，按卖出比例平仓

### 需求:系统必须支持均值穿越策略回溯
系统必须支持基于价格穿越均线的策略回溯。

#### 场景:配置均值穿越策略
- **当** 用户选择"均值穿越策略"
- **那么** 系统显示参数配置：均值周期、买入比例、卖出比例、初始资金
- **当** 用户配置参数并运行回溯
- **那么** 系统根据历史数据模拟交易

#### 场景:均值穿越买入信号
- **当** 收盘价从下方穿越均线
- **那么** 系统触发买入信号，按买入比例建仓

#### 场景:均值穿越卖出信号
- **当** 收盘价从上方穿越均线
- **那么** 系统触发卖出信号，按卖出比例平仓

### 需求:系统必须展示完整的回测结果
系统必须展示收益、风险、交易全量指标。

#### 场景:收益指标展示
- **当** 回测完成
- **那么** 系统展示：累计收益率、年化收益率、总盈亏金额

#### 场景:风险指标展示
- **当** 回测完成
- **那么** 系统展示：最大回撤、最大回撤持续时间、波动率

#### 场景:交易指标展示
- **当** 回测完成
- **那么** 系统展示：交易次数、胜率、平均持仓天数、盈亏比、夏普比率

#### 场景:基准对比
- **当** 回测完成
- **那么** 系统展示同期买入持有策略的收益率对比

### 需求:系统必须展示交易明细
系统必须展示每笔交易的详细信息。

#### 场景:交易明细列表
- **当** 回测完成
- **那么** 系统展示交易明细列表，包含：日期、类型、价格、比例、金额、持仓数量、盈亏
- **当** 用户点击某笔交易
- **那么** 系统高亮显示该交易在图表上的位置

### 需求:图表必须标注买卖信号
系统必须在图表上标注买卖信号点。

#### 场景:信号点标注
- **当** 回测图表渲染
- **那么** 系统在买入信号位置标记向上箭头
- **当** 卖出信号位置
- **那么** 系统标记向下箭头
- **当** 用户悬停信号点
- **那么** 系统显示该笔交易的详细信息

### 需求:用户可以选择回测时间范围
系统必须支持自定义回测时间范围。

#### 场景:时间范围选择
- **当** 用户配置回测参数
- **那么** 系统提供时间范围选择器
- **当** 用户选择时间范围
- **那么** 回测仅使用该时间段内的数据

### 需求:系统必须提供默认策略参数
系统必须为新策略提供合理的默认参数。

#### 场景:均值偏差默认参数
- **当** 用户创建新的均值偏差策略
- **那么** 系统预设：均值周期60日、买入阈值-5%、卖出阈值5%、买入比例100%、卖出比例100%、初始资金100000

#### 场景:均值穿越默认参数
- **当** 用户创建新的均值穿越策略
- **那么** 系统预设：均值周期60日、买入比例100%、卖出比例100%、初始资金100000
